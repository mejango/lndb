## Vision

### 10x Check

The current plan is a good website. The 10x version is a **digital-to-physical bridge** — the website doesn't just tell you about trees, it begins a relationship with a tree that eventually leads you to a place where that tree lives (the resort).

Concretely:
- Tree pages designed as **living portals**, not static brochures — expandable over time with seasons, rituals, community stories
- The meditation is a taste of an **in-person experience** — "Visit this tree in the forest"
- The arboleda (personal forest) becomes something you can **physically gift** — a framed poster, a tree planting, a resort booking
- The quiz becomes the **sorting hat** for the resort — "You're a Ceiba. Your cabin is this way."

### Platonic Ideal

You open the website. The screen is dark. A forest sound fades in. Slowly, leaves part, and you're inside the forest. You see the 16 trees, distant but alive. You tap one — it comes closer. You hear its voice whisper its name. You feel like you've entered a sacred place.

You take the quiz. It's not a questionnaire — it's a conversation with the forest. "The forest has been watching you. It knows which tree you are." When you get your result, it doesn't feel like a personality test — it feels like recognition. "You've always been a Ceiba. You just didn't know your name."

You share your tree with someone you love. Not a link — a message that feels like a gift. "I found your name in the forest. You're a Guayacan Rosado."

The meditation plays. You close your eyes. When you open them, there's a gentle invitation: "Your tree is waiting for you in the real forest. Come meet it."

**The feeling:** Awe, recognition, belonging. The brand's Mapa Sagrado nails this — "magical, wise, and profoundly human."

## Implementation Decisions Resolved

- **Email capture service:** Formspree (free tier, 50 submissions/month). When volume exceeds 50/month, upgrade to paid tier ($8/month for 1000 submissions) or migrate to Netlify Forms.
- **Book purchase link:** Instagram DM link (pre-filled message with tree name)
- **Header/footer DRY:** Fetch-and-inject pattern via `js/components.js` — each page has `<div id="header"></div>` placeholder, JS fetches shared HTML fragments and injects them. No build step. No framework.
- **Tree data:** Extract `trees` object, `descriptions` object, and asset path mappings from `quiz.html` into a shared `js/trees.js` file. Both quiz.html and all 16 tree pages import this single data source.
- **Audio hosting:** Static in repo for v1 (meditation audio files in `assets/audio/meditations/`, ambient loops in `assets/audio/ambient/`). Audio files should be compressed MP3, max 5MB per meditation, max 500KB per ambient loop. If repo exceeds 200MB total, migrate audio to a CDN.
- **Ambient soundscapes:** Creative Commons ambient audio (e.g., freesound.org), different per tree biome. Looping MP3 files, ~30 seconds each, under 500KB. Trees grouped by biome: mountain (Palma de Cera, Roble), tropical lowland (Mango, Cacao, Tamarindo), temperate forest (Ceiba, Nogal, Eucalipto), flowering (Guayacan R, Guayacan A, Cerezo, Arrayan), dry (Cenizo, Copal), cloud forest (Yarumo, Caimito).
- **Approach:** B (The Forest Experience) — 16 static HTML files, one per tree. Static HTML required for OG meta tags (social crawlers don't execute JS). Each file is a thin shell with tree-specific `<meta>` tags; all logic lives in shared JS files.
- **Build/tooling:** No build step. No bundler. No package.json. Pure vanilla HTML/CSS/JS. Development requires a local HTTP server (`python -m http.server` or similar) because `js/components.js` uses `fetch()` for header/footer injection, which is blocked on `file://` due to CORS.
- **Tree page content:** Descriptions reused from `quiz.html`'s existing `descriptions` object (already written for all 16 trees). Background photos from `assets/images/backgrounds/`. Illustrations from `assets/images/trees/`. No new content creation needed except microcopy for UI elements.
- **"Coming soon" state:** Data-driven, not runtime. `js/trees.js` data includes `hasMeditation: true/false` per tree. If false, show "Meditation coming soon" with the tree's illustration and description still visible. Do NOT use 404 detection — a CDN glitch or typo would falsely show "coming soon."
- **Share card specs:** Two canvas-generated formats: (a) 1080x1920px Instagram Stories — tree illustration centered, name in Isaodelco font, short description below, brand logo at bottom; (b) 1200x630px WhatsApp/link preview — tree illustration left, name + description right, brand bar at bottom. On mobile, use Web Share API (`navigator.share({ files: [pngBlob] })`); on desktop, show image inline with download button.
- **Naming nudge behavior:** Gentle semi-transparent overlay that slides up from bottom of tree page after 15 seconds. Text: "Conoces a alguien que sea este arbol?" with two buttons: "Tomar el quiz para alguien" (links to quiz) and "Compartir" (triggers share card). Dismissible. Does not appear if user arrived from quiz (check URL param `?from=quiz`). Uses sessionStorage to avoid re-showing after dismissal.
- **Name inscription animation:** Fade-in typewriter effect. User's name appears letter by letter above the tree name: "Sofia," (pause) "eres Ceiba." Duration: ~2 seconds. Uses CSS animation, no library needed.
- **Meditation completion ritual:** When audio `ended` event fires, fade in an overlay with: (a) a brief message in the tree's voice (pre-written in `js/trees.js`, e.g., "Gracias por escucharme. Lleva este momento contigo."), (b) a share button, (c) a "next meditation" link to the next tree in book order (using `treeOrder` array, wrapping from Copal to Yarumo). Overlay is dismissible. No focus trap — uses ARIA `role="status"` and Escape-to-dismiss.
- **Naming:** Use "Copal" consistently (not "El Copal") across all pages. During data extraction from quiz.html to js/trees.js, rename the `name` field from 'El Copal' to 'Copal'.
- **Background filename mapping:** Background photos use inconsistent naming (e.g., `Nogal_Cafetero 2.jpg`, `Caimito_ 2.jpg`, `Guayacan_A. 2.jpg`). The `js/trees.js` data file must include explicit `backgroundFile` paths per tree. Refer to the design doc's Asset Mapping table for the complete tree-to-filename mapping.
- **Audio hosting clarification:** The repo's tracked file size is already significant due to high-res source PNGs. For v1, meditation audio (16 x max 5MB = 80MB) and ambient loops (16 x max 500KB = 8MB) go in the repo. If the working tree exceeds 300MB of tracked files, migrate audio to Git LFS or external CDN. Consider adding a `.gitignore` for the source asset directories (`PNG_Arboles/Alta/`, `Fondos/`, `Tipografias/`) which are not needed for the web build.
- **Canvas font loading:** Use `FontFace` API with `document.fonts.add()` and `await document.fonts.ready` before drawing text on canvas. Specify a CSS fallback font stack (e.g., `Georgia, serif`) for canvas rendering in case Isaodelco fails to load.

## Implementation Decisions (from Eng Review)

- **Local dev server required:** `python -m http.server` or equivalent. `file://` does not work due to `fetch()` CORS in `js/components.js`.
- **Coming soon = data-driven:** `hasMeditation: true/false` in `js/trees.js`, not runtime 404 detection.
- **Analytics = Plausible:** Cookie-free, no consent banner, lightweight. Events: quiz_complete, tree_page_view, meditation_play, meditation_complete, share_card_generated, email_captured.
- **OG images = one-time script:** `tools/generate-og.js` (Node + Canvas), run manually, commit PNGs to `assets/og/`.
- **Image optimization = one-time script:** `tools/optimize-images.js` generates compressed web versions of tree illustrations. Progressive loading: bg photo first, illustration fades in.
- **Naming nudge + completion ritual:** NOT modals. No focus trap. Escape-to-dismiss via keydown listener. ARIA `role="status"` for screen readers.
- **Ambient audio autoplay:** Attempt muted autoplay. If blocked by browser policy, show a subtle pulsing sound icon inviting the user to tap. Silent failure is not acceptable.
- **Homepage meditation preview:** Play first 30s of full meditation file using JS `timeupdate` event to pause at 30s. No separate preview audio files.
- **Asset mapping = in js/trees.js:** Each tree entry includes explicit `illustration`, `backgroundFile`, and `slug` fields with exact filenames. Single source of truth. Stray `Baobab 3.jpg` in backgrounds/ is ignored.
- **Canonical tree order (from book):** Yarumo, Guayacan Amarillo, Cenizo, Nogal Cafetero, Caimito, Roble, Tamarindo, Cerezo, Palma de Cera, Arrayan, Ceiba, Guayacan Rosado, Mango, Cacao, Eucalipto, Copal. Explicit `treeOrder` array in `js/trees.js`. Prev/next navigation uses this array. Wraps from Copal → Yarumo.
- **Tablet breakpoint (768-1024px):** Content panel at 80% width, 2-column forest grid, intermediate font sizes.
- **Mobile share fallback:** If `navigator.share()` rejects or is unavailable, fall back to showing image inline with download link (same as desktop path).
- **Formspree error handling:** Button disabled + spinner during submission. Honeypot field for spam. 429/quota errors show "No se pudo enviar. Intenta de nuevo." Duplicate submissions prevented by disabling button after success.
- **Keyboard seek (±10s):** This is an accessibility feature, not a visible scrubber. No contradiction with "no fast-forward" UI design.
- **Testing strategy:** `/qa` browser-based testing via Playwright MCP after implementation. No test framework added (matches no-package.json constraint).

## Visual Design Specifications (from /plan-design-review)

### Tree Page Layout — Desktop (> 1024px)

Background photo fills entire viewport (100vh, object-fit: cover, fixed position).
Dark gradient scrim (forest green #3B4A3A at 40% opacity) at bottom of hero viewport,
transitioning into cream content panel.

Hero zone (above scroll):
- Sound toggle: top-left, 44px touch target, sage green icon
- Tree navigation arrows: top-right
- Tree illustration: centered, ~40vh height
- Tree name: Isaodelco 48px, centered below illustration
- Scroll indicator: subtle animated arrow

Content panel (scrollable, below hero):
- Background: rgba(245,237,224, 0.92), border-radius 16px top only
- Max-width: 640px, centered
- Padding: 32px
- Content order: description → divider → meditation player → divider → resort seed → divider → gift CTA → prev/next trees

### Tree Page Layout — Mobile (< 768px)

Same structure but:
- Tree name: Isaodelco 36px
- Content panel: full-width, padding 24px
- Meditation player: full-width
- Gift CTA: full-width button
- Email field + button: stacked on very small screens

### Meditation Audio Player

Minimal and organic — part of the page texture, not a widget.
- Progress: thin 1px horizontal line, sage green base, fills with coral as audio plays
- Play/pause: single circular button, 48px, sage green background, leaf-like play icon
- Time display: Mulish Light 12px, muted color, right-aligned
- No volume control (system volume), no fast-forward, no scrubber drag
- Focus ring: 2px coral outline for keyboard nav
- ARIA: aria-label="Meditacion de [tree name]"

### Forest Grid (Homepage)

Organic feel — trees overlap slightly (negative margin -8px on alternating rows).
- 4 columns desktop, 2 columns mobile
- Each cell: tree illustration on cream bg, no visible border
- Name below: Mulish Regular 14px, forest green
- Hover: scale(1.05) + faint background photo crossfade behind illustration
- Gap: 16px desktop, 12px mobile

### Homepage Meditations Teaser Section

Section heading: "Meditaciones del Bosque" (Isaodelco)
Subtitle: "Escucha la voz de cada arbol" (Mulish)
Show 3 trees with recorded meditations, each with:
- Inline mini-player (same style as tree page player)
- 30-second preview clip
- Tree name + description excerpt
- Text link: "Escuchar meditacion completa" → tree page

### Quiz Result Screen

- Tree illustration: centered, ~200px
- Typewriter animation: "Sofia," (80ms/char, pause 500ms) "eres Ceiba." (80ms/char)
- Isaodelco large (36px mobile, 48px desktop)
- Description excerpt: Mulish Light 16px
- CTAs: "Escuchar tu arbol" (btn-primary), "Compartir" + "Nombrar a alguien" (btn-outline)
- Transition to tree page: fade-in animation (opacity 0→1 over 1.5s) via ?from=quiz param

### Naming Nudge Visual Treatment

NOT a card/container/modal. Text over gradient.
- Bottom of viewport, gradient fades from background photo to semi-transparent cream
- Text: Mulish Italic, sage green, centered
- Buttons: text links (btn-text style), not boxed buttons
- Entry: slide-up 0.6s ease-out
- Dismissal: slide-down 0.3s ease-in
- Feel: handwritten marginalia, not a popup

### Meditation Completion Ritual

NOT a modal with dark backdrop.
- Audio fades out. 3-second pause of intentional stillness.
- Tree's message fades in: Mulish Italic, sage green, centered on page
- After 1-second delay: share button + "next meditation" link fade in below
- No box, no card — text floating on the existing page
- Dismissible by scrolling or tapping anywhere

### "Coming Soon" Meditation State

Same tree page layout — BG photo, illustration, name, description all present.
Where player would be:
- Tree illustration at 60% opacity
- "Esta meditacion llegara pronto" (Mulish Italic)
- "Mientras tanto, cierra los ojos e imagina el sonido de este arbol." (warm invitation)

### Typography Scale

- Tree name (hero): Isaodelco 48px desktop / 36px mobile
- Section headings: Isaodelco 32px desktop / 24px mobile
- Body text: Mulish Regular 400, 16px, line-height 1.7
- Poetic descriptions: Mulish Light 300, 18px, line-height 1.8
- Microcopy/labels: Mulish Medium 500, 13px, uppercase, letter-spacing 1px
- Button text: Mulish SemiBold 600, 14px

### Animation Vocabulary

- fade-in: opacity 0→1, 0.8s ease
- slide-up: translateY(20px)→0 + fade-in, 0.6s ease-out
- dissolve: page transitions, 1.5s ease
- typewriter: letter-by-letter, ~80ms/char
- scale-hover: transform scale(1.05), 0.3s ease
- All animations respect @media (prefers-reduced-motion: reduce)

### Accessibility

- Skip link: "Ir al contenido" on every page
- Audio player: keyboard Space/Enter for play/pause, arrow keys ±10s
- Overlays (naming nudge, completion ritual): focus trap, Escape to dismiss
- All interactive elements: 44px minimum touch target
- Tree illustrations: alt="Ilustracion del arbol [name]"
- Form fields: <label> elements, aria-required for email
- Contrast: gradient scrim ensures text readability on all background photos
